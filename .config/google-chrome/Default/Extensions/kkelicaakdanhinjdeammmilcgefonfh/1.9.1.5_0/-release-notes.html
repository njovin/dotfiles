A simple explanation for non-developers is this:
Basically, any extension that is able to interact with a web page by reading or modifying it's content, is able to read anything you enter in web forms, from usernames & passwords to credit card details and other sensitive data (basically, any text displayed by your browser - even the stuff you type in password fields). The thing is that most extension actually need the ability to interact with pages for legit reasons and it's hard to decide which you should trust and which not. As an example from the top of my head, all ad blockers, color pickers, extensions that enable mouse gestures, form auto-completers, password managers (doh!), text highlighters (and the list could go on) have the technical capability to silently steal your data without you even suspecting it, if their developer would choose to do so.

As for developers:
Extensions interact with webpages through "content scripts" which are plain JavaScript injected into every page that loads in the browser. This means that the extension can read the DOM and attach keypress event handlers to anything and send the captured data anywhere using AJAX calls (yes, extensions can make cross-domain AJAX calls).

(In the case of Window Resizer, a content script is used to create the tooltip shown when you resize the window.)

I know this is really scary stuff; it's the same thing I fear every time I install a new extension myself. I just didn't wanted you to have these fears when it comes to Window Resizer; that's what I meant in my previous notes.

Once again, I'm sorry if I've disappointed you in any way! I hope that clears up the things a bit, but, if you have any other concerns, feel free to contact me using the Support link in the webstore or by sending an email at ionut.botizan@gmail.com, with the subject "Window Resizer"

Thank you!

Version 1.6.1 (2011-03-30):

  • Improved mechanism for resizing the viewport. Previously, if the window was maximized when trying to resize the viewport, there was a bug that caused the extension to set negative values for the width and height, and the window ended being 0 pixels wide.
  • The extension now needs the permission to access the content on all pages. This is required in order for the extension's option to resize just the viewport to work in the latest Chrome dev releases.

    Technical info:
    In order to resize the viewport, the extension needs to know the size of the window's borders, but, unfortunetly, this isn't available through the API. So, in order to determine the borders' size, the extension reads the width and height of the actual window, then it takes a screenshot of the visible area of the current tab and it substracts the size of the generated image from the window's size.
    Unfortunetly, the latest dev versions of Chrome require the extension to ask for permission to access the content of any given tab in order for it to be able to take a screenshot of that tab using the API and that's why it stopped working.
